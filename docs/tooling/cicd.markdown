## CI/CD

During the development of UI5-Application, some tasks have to be carried out regularly.

- Build
- Linting
- Testing and
- Deployment

of a UI5-Application characterice the common life cylce.

These tasks can be carried out manually using the terminal, cmd or the UI of a code-editor.
On the other hand, there are some reasons to automate and standardize these tasks:

- a growing number of supported UI5-Applications
- increasing complexity of the executed tasks
- occuring mistakes while executing tasks manually
- forgotten execution of tasks
- needed time for execution and waiting period of the tasks

One way to automize these tasks is using a CI/CD Pipeline. CI stands for Continuous Integration / Continuous Delivery and can widely be described by the 4 tasks mentioned in the beginning.
